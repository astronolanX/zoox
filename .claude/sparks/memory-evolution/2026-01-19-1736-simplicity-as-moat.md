## Insights

The contrarian inversion cuts deep[^1]: simplicity-as-moat assumes attackers want in. But what if emptiness repels *users*, not just complexity? The blank book metaphor[^2] reveals the paradox: reef's value is projection. Users fill emptiness with meaning — but they can also fill it with nothing.

The self-elimination insight[^3] connects to earlier thread findings: reef succeeds by becoming unnecessary. But planned obsolescence as defensive strategy inverts typical product thinking. Most products fight to become indispensable; reef fights to become invisible.

**The vulnerability:** emptiness provides no surface for attachment. Features are dependency hooks. Zero features means zero lock-in — which means users can leave without cost. The moat works both ways.

**Heretical direction:** What if reef should add *exactly one* feature that creates healthy dependency? Not feature creep, but strategic stickiness. The question: what's the minimum viable hook?
